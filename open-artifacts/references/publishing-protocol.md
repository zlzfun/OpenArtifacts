# Publishing Protocol

Read `config/open-artifacts.toml` for server configuration.

The publish helper supports only these client-side environment overrides:
`OPEN_ARTIFACTS_SERVER_URL`, `OPEN_ARTIFACTS_API_BASE`,
`OPEN_ARTIFACTS_ORGANIZATION`, `OPEN_ARTIFACTS_WORKSPACE`,
`OPEN_ARTIFACTS_DEFAULT_VISIBILITY`, and `OPEN_ARTIFACTS_PUBLISH_TOKEN`. Do not
scan or guess other environment variable names.

Prefer passing payload JSON on stdin with `--payload -`. If the runtime cannot
pipe stdin reliably, write any temporary payload file inside the current
workspace or the runtime's normal temp directory. Do not create root-level
temporary directories such as `C:\Temp` or write under sensitive system paths.

PowerShell example:

```powershell
@'
{
  "schema_version": "0.1",
  "title": "Example",
  "kind": "work-summary",
  "summary": "Published from stdin.",
  "status": "draft",
  "workspace": {"name": "default"},
  "source": {"agent": "opencode"},
  "blocks": [{"type": "markdown", "title": "Summary", "content": "Hello"}]
}
'@ | python open-artifacts/scripts/publish_artifact.py --config open-artifacts/config/open-artifacts.toml --payload - --idempotency-key example-1
```

Publish endpoint:

`POST {server_url}{api_base}/artifacts/publish`

Headers:

- `Authorization: Bearer <publish_token>`
- `Content-Type: application/json`

Request:

```json
{
  "artifact_id": "optional-existing-id",
  "idempotency_key": "unique-key",
  "visibility": "private",
  "payload": {}
}
```

Response:

```json
{
  "artifact_id": "art_123",
  "version": 1,
  "url": "http://localhost:8787/a/art_123",
  "gallery_url": "http://localhost:8787/gallery"
}
```

On `401`, stop immediately and report invalid Skill/server token configuration.
Do not retry without a token, guess common passwords, brute-force tokens, or
probe environment variables. Ask the user for the correct publish token or for
confirmation that `open-artifacts/config/open-artifacts.toml` has been updated.

On `422`, revise the payload according to the validation detail. If the detail
mentions `Unsafe SVG`, remove the named disallowed tag or attribute. Retry at
most once with SVG; if it still fails, replace the SVG with structured blocks
such as `flow`, `table`, `chart`, `stat-grid`, or `callout`.

On network failure, preserve the generated payload and report publish failure.

The returned `url` and `gallery_url` are generated by the server from
`OPEN_ARTIFACTS_PUBLIC_BASE_URL`. A client-side `server_url` controls where the
publish request is sent, but it does not configure the public viewer URL. For an
nginx deployment on a non-default port, set server-side `PUBLIC_BASE_URL` or
`OPEN_ARTIFACTS_PUBLIC_BASE_URL` to include the port, for example
`http://203.0.113.10:8788`.
